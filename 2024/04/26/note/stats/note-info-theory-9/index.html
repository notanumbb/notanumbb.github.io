6.8 universal portfolio

回忆前面，在已知 X∼FX\sim FX∼F 的情况下，我们可以通过最大化 W(b,F)W(b, F)W(b,F) 来找到一个 log-optimal portfolio b∗b^*b∗；但问题是我们往往不知道股票的分布 FFF 具体是啥，也就是说对于任意的给定的 bbb，我们无法计算 W(b,F)W(b, F)W(b,F)，也就无法评估 / 比较这个 portfolio bbb 的好坏；

换一个思路呢：假设当前有一个 portfolio bbb，虽然不知道 FFF 具体是啥，但是我们找一个使得当前的 portfolio 表现最烂的一组 nnn 个 mmm 维向量，代表当前投资组合下的 worst case，并计算它在 worst case 下的表现，来作为这个 portfolio 的评价指标；

但是考虑到不同的 worst case 烂的程度不一，我们这里引入一个 constant rebalance portfolio 来作为某种 "标准化"：前面已经证明了在已知 FFF + iid market 的情况下，log-optimal portfolio b∗b^*b∗ 作为一组常数是渐进最优的，则对于当前已经给定的 n 天结果 (x1,...,xn)(x_1, ..., x_n)(x1​,...,xn​)，我们针对这个结果计算一个最优的 constant portfolio (要求 n 天都一样的投资组合)，记作 constant rebalance portfolio，并用它的表现来标记 "在当前的 worst case 下能够达到的某种 optimal 表现"；再用当前的 portfolio bbb 和这里的 constant rebalance 比较，看我们的 bbb​ 在 worst case 下会比 optimal 差多少；

用 nnn 个时间点的 nnn 个 m 维向量 x1,...,xnx_1, ..., x_nx1​,...,xn​ 来表示当前 portfolio 的 worst case，并基于它找到一个表现最好的 constant portfolio；记我们的 constantly rebalanced portfolio 所能达到的最佳收益为 Sn∗S_n^*Sn∗​，
