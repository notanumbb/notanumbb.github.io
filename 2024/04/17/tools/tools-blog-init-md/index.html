hexo + GitHub Page 搭建博客站点

发表于2024-04-17|toolsblog

记录本站点搭建基本流程



1 hexo 配置与部署

安装 Hexo

首先检查配置

  • Git
  • node.js

Hexo 是基于 Git 与 Node.js 运行的；基本环境搭建结束后，可以使用 npm 命令进行安装

进入博客文件夹，进行初始化

检查指定的文件夹目录如下

  • .gitignore：大家的好朋友 gitignore

  • _config.yml：网站配置信息，这是 Hexo 的 config 文件

  • _config.themename.yml：主题配置文件；init 的时候会初始化一个叫 landscape 的主题；

  • package.json：基本安装包与渲染引擎；EJS 和 markdown 都默认安装

  • /scaffolds：模板文件夹；内部放 .md 文件，新建文章中会默认填充模板的内容

  • /_source：用户文件夹

    • /_drafts：草稿文章，不会被渲染到 html 里；
    • /_post：正常可以发布的文章，以多个 .md 的形式存储，内部的文件会自动被渲染

  • /theme：主题文件夹，内含 html 文件渲染时的样式主题信息；

  • /public：执行第一次渲染后，会生成 public 文件夹；里面存储生成的静态网页文件

2 基本 hexo 命令

hexo new [layout] <title>

新建一篇名为 tilte.md 的文章 (生成在 source/_post/ 文件夹下)；layout 为模板名称，可选 (模板存储在 scaffolds/ 下)

hexo clean

清空 public 文件夹

hexo g

也就是 generate；在 public 下渲染得到静态网页

hexo s

部署到本地，可以在 http://localhost:4000/ 查看预览的网页 (具体端口号注意提示信息

hexo s --debug

以调试模式部署，此时随时更改主题文件等再刷新页面可以马上看到效果

hexo d

部署到服务器端 (比如这里的 GitHub Page)

hexo new page pagename

创建新页面，此时会自动在 source 下生成对应的文件夹，并内部生成默认的 index.md 文件；编辑文件进行内容展示；



3 picGo + Github 搭建图床

Github 自建 public repo；

picGo - 图床设置 - Github

  • 仓库名：username/reponame
  • branch：默认 main
  • 设定 Token：Github - setting - 开发者选项部分，新建 Token，权限勾选 repo 的部分就行，expiration 最好选永久；记下来粘贴进来
  • 存储路径：设定后会自动在 repo 下建立对应的路径，以后都传输到这里；

然后去 typora 设定服务；建议默认选择复制到 ./assets/ 文件夹，需要上传到博客的文章再手动右键选择上传，否则自己随便写写的都上传了会很麻烦 ...

个人自用的几个：

  1. markdown-it-footnote

    内置 plugin，不需要再重复安装；支持脚注语法 [^1]

  2. markdown-it-mark

    内置 plugin，不需要再重复安装；支持高亮语法 ==heightlight==

  3. markdown-it-task-lists

    支持 GMD 风格的 checklist；详见 markdown-it-task-lists

  4. markdown-it-github-alerts

    支持 GMD 风格的 alert 警告，形如 > [!note]

  5. '@renbaoshuo/markdown-it-katex'

    hexo-renderer-markdown-it 官方推荐的 katex 编译；按照教程安装就行，注意要在 post 前通过 html 语法引入 css，建议直接写到 template 里面这样每次新建的时候都会在。

    其实个人体感 Mathjax 3.0 的速度和 katex 已经相差不大了，但是前者不支持 \\ 换行 (一概会被 html 转义为 \)，平常用 Typora 写作搞这出实在是受不了，对比官方 issue 提出这个问题的人也不少，挺好奇为啥就是一直没有得到解决。
