3.2 指针 (pointer)

一句话总结：指针是一种变量，这个变量的值是另一个变量的地址；

为了区分指针变量和其他的普通变量，我们在指针变量的变量名前面加上一个 *


指针的声明和定义

类似变量，指针也需要定义 / 声明：

进行定义时：type* p

  • 这里的 * 表示定义的变量是一个指针；
  • type 为指针指向的变量的类型，需要是一个有效的 c++ 数据类型；

进行赋值时：p = &var；这里直接对 p 赋值，右侧是 var 变量的地址；

重复：指针变量的值是另一个变量的地址；我们通过 &varname 来获取一个变量的地址；

  • *p：访问 (指针变量 p 所存储的地址) 对应的值，所以此时的输出为 var 变量的值

指针的运算与比较

我们可以对指针进行四种算术运算：++, --, +, -


  1. 指针的算术运算

这里的 +, - 表示从指针当前位置出发，往后或往前进行移动；具体移动的步数为指针对应的数据类型大小；比如存在指向整型变量的指针 int *p，则执行 p++ 后指针会后移 4 字节 (32 位整型一个占用 4 字节)

总结为：

  • 加法运算：当一个指针 p 加上一个整数 n 时，结果是指针 p 向后移动 n 个元素的大小。例如，如果 p 是一个 int 类型的指针，每个 int 占 4 个字节，那么 p + 1 将指向 p 所指向的下一个 int 元素。
  • 减法运算：当一个指针 p 减去一个整数 n 时，结果是指针 p 向前移动 n 个元素的大小。例如，如果 p 是一个 int类型的指针，每个 int 占 4 个字节，那么 p - 1 将指向p所指向的前一个 int 元素。

3.3 引用变量 (reference)

一句话总结：引用变量是一个别名，也就是说，它是某个已存在变量的另一个名字

我们可以将变量名看成是内存中一块空间的 "标签"，而变量的定义也就是在内存中开辟一个 type 大小的空间，并为它贴上变量名这个标签，从此程序中可以通过这个标签找到对应的内存空间，从而读取中间的值；

而引用变量可以理解为，我们为同一块内存空间取了一个别名，则这一块内存会同时存在多个标签，原变量名和引用名都可以用来访问这个空间和它中间存储的值；

这里和普通变量的区别在于：

  • 普通变量 int a = b;：等价于在内存空间中新开辟了一块大小为 int 的地址，取名为 a，并将已经存在的名为 b 的空间内的值复制一份，填入到新的地址中；此时对 a 的修改自然也不会再涉及 b；
  • 引用变量 int& a = b;：等价于先找到已经存在的名为 b 的内存空间，再给它一个别名 a；则此时对 a, b 的访问 / 修改其实都是作用在同一块内存空间上，互相波及；

引用 vs. 指针

引用和指针存在不同：

定义上：指针和引用都是一个新的变量

  • 指针的变量值是另一个变量的地址；此时 ptr 输出的为地址，赋值时右值需要为一个变量的地址 int* ptr = &var，且需要通过 *ptr 来访问对应地址内的值；
  • 而引用是另一个变量对应的内存空间的 "别名"，赋值时直接等于原始的变量名 int& ref = var; 而直接通过引用 ref 和原本的变量名 var 都可以访问到同一块内存空间内的值；

初始化上：

  • 指针可以初始化为空指针，也就是默认指向地址 0，规定为不指向任意对象；
  • 而引用初始化时不允许为空，必须在定义时就给定一个已经存在的内存空间变量名；

更改上：

  • 指针可以随时更改；本质上指针变量的值也就是另一个变量的地址，则通过修改指针变量的值可以让指针指向不同的内存空间 / 变量；
  • 引用不允许修改，一旦引用被初始化为一个对象 (成为某一块内存空间的别名)，就不能再被指向到另一个对象；
