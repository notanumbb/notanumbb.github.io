Note

建议在函数声明时再指定参数的默认值；

  • 对于 g++ 编译器，声明和定义时都可以指定参数的默认值；
  • 对于 MSVC 编译器，如果声明存在则只允许在声明时指定默认值，不允许在定义时给定；
  • 大部分情况函数定义的文件不会对外展示，在对方只看到函数声明的情况下，建议全部在声明的时候进行默认值的指定


4.2 函数参数

函数使用参数，则必须声明接受参数值的变量。这些变量称为函数的 形式参数；形式参数就像函数内的其他局部变量，在进入函数时被创建，退出函数时被销毁

调用函数时，有三种向函数传递参数的方式：


传值调用

也就是把参数的实际值复制给函数的形式参数；在这种情况下，修改函数内的形式参数不会影响实际参数。

传值调用是 C++ 默认的函数参数传递方法；

实际上执行时，是先创造 x, y 两个新的参数，再将对应的 a, b 的值赋给 x, y，再对 x, y 进行函数操作；执行结束后，形式参数 x, y 被销毁，也无法再访问；


指针调用

也就是把参数的地址复制给形式参数。在函数内，该地址用于访问调用中要用到的实际参数。这意味着，修改形式参数会影响实际参数；

  • 此时的函数参数需要被声明为指针类型；
  • 调用函数时传入的是对应变量的指针

引用调用

引用调用中，把引用的地址复制给形式参数。在函数内，该引用用于访问调用中要用到的实际参数。这意味着，修改形式参数会影响实际参数。

  • 此时需要将形式参数定义为引用类型
  • 传入参数时直接传入原变量

引用传递避免了传递大对象时的拷贝开销，提高了效率；类似地，我们也可以将返回值在外部定义好，再通过引用传递进入函数，直接通过函数修改外部定义好的返回值，同样可以避免返回值拷贝的开销
