C++ 在 C 的基础上增加了面向对象编程；

类可以理解成一种用户自定义的数据类型，内部包括类变量和类函数，类可以被看作是一种模板，可以用来创建具有相同属性和行为的多个对象；

  • 类的理解与基本操作
  • 类的访问修饰符
  • 类的几个特殊函数
  • 派生类与继承


5.1 类的基本操作


类的定义

使用 class 关键字来定义一个类；

  • class 关键字，表示开始定义一个类；
  • access_specifiers：访问修饰词，规定成员的可见性和区域权限
  • 类的主体包括：
    • 类变量，这里用变量声明一样的方式来进行声明，可以给定初始值 (可选)
    • 类函数，也就是一些类内部带有的方法；
      • 可以在类的内部进行声明，然后在类的外部定义函数，使用 class_name::func 的形式来进行
      • 也可以直接在类的内部进行定义；
  • 注意类的最后需要一个分号表示结束；

5.2 类的访问修饰符 + 静态

数据封装是面向对象编程的一个重要特点，它防止函数直接访问类类型的内部成员。类成员的访问限制是通过在类主体内部对各个区域标记 public、private、protected 来指定的，称为访问修饰符。

一个类可以有多个 public、private、protected 标记区域。每个标记区域在 1) 下一个标记区域开始之前；2) 遇到类主体结束右括号之前；都是有效的。

成员和类的默认访问修饰符是 private


公有成员 public

公有成员在程序中类的外部是可访问的；也就是说在类定义的外部，进行实例化得到一个对象后，可以通过 . 来访问任意的一个公有成员 (变量或函数)

一般来说，类的函数 (对外) 都是公有成员，用 public 修饰，以便在实例化后仍可以通过对象来调用这些函数


私有成员 private

私有成员变量或函数在类的外部是不可访问的，甚至是不可查看的；

  • 类定义的内部可以访问 / 修改，包括类外面定义的类成员函数；
  • 利用类 public 公有成员函数可以输出 / 查看私有成员

默认情况下，类的函数 / 变量是私有成员

一般来说类内部使用的数据都是私有成员，使用 private 修饰；


受保护成员 protected

受保护成员不可在类的外部进行访问 / 查看 (和 private 相同)，但是在派生类 (子类) 中是可以访问的；


类的静态成员 static

可以使用 static 关键字来把类成员定义为静态的。当我们声明类的成员为静态时，这意味着无论创建多少个类的对象，静态成员都只有一个副本

静态成员在类的所有对象中是共享的；换言之即使没有创建对象，静态数据成员也存在，且在程序的整个生命周期内存在，直到程序结束

静态成员需要在类的内部进行声明，但在类的外面进行定义和初始化 (为了保证空间的分配)

类似地，类也可以有 static 修饰的静态函数

静态成员函数即使在类对象不存在的情况下也能被调用，静态函数只要使用类名加范围解析运算符 :: 就可以访问

  • 静态函数可以类似普通成员函数，在类内部 / 外部定义；
  • 静态成员函数只能访问类的静态成员 (静态成员数据、其他静态成员函数) 和类外部的其他函数；而其他的成员函数是可以正常访问静态成员的；
  • 静态成员函数不能使用类的 this 指针

析构函数

类的析构函数是类的一种特殊的成员函数，它会在每次删除所创建的对象时执行。

析构函数的名称与类的名称是完全相同的，只是在前面加了个波浪号（~）作为前缀，它不会返回任何值，也不能带有任何参数；析构函数有助于在跳出程序（比如关闭文件、释放内存等）前释放资源

友元函数 / 类

类的友元函数是定义在类外部，但有权访问类的所有私有（private）成员和保护（protected）成员；

  • 尽管友元函数的原型需要在类的定义中出现，但是友元函数并不是成员函数；也就是说调用友元函数的时候是直接用名字的函数调用方式，且不能使用成员函数中的一些设定 (比如 this.)
  • 友元可以是一个函数，该函数被称为友元函数；友元也可以是一个类，该类被称为友元类，在这种情况下，整个类及其所有成员都是友元。

如果要声明函数为一个类的友元，需要先在类定义中该函数原型前使用关键字 friend，再在外部定义函数体



5.4 派生类与继承

面向对象程序设计中最重要的一个概念是继承。

继承允许我们依据另一个类来定义一个类，这使得创建和维护一个应用程序变得更容易。这样做，也达到了重用代码功能和提高执行效率的效果；

在支持继承的语言中，我们可以指定新建的类继承了一个已有的类的成员。这个已有的类称为基类 (base)，新建的类称为派生类 (derived)

这里的派生类可以理解成基类的一个特例，派生类有基类的性质，同时有一些自己特别的特征；继承代表的是一种 is a 关系，例如狗是动物的一种，则 animal 就是基类，而 dog 是继承自 animal 的一个派生类；


派生类定义

通过 : 来指定一个派生类

其中：

  • BaseClass 是父类或基类；一个类可以派生自多个类，这意味着，它可以从多个基类继承数据和函数；
  • DerivedClass 是子类或派生类；
  • access_specifier 可以是 public、protected 或 private，指定了派生类对基类成员的访问权限；如果这里不指定，默认为 private。

注意派生类可以访问基类中所有的非私有成员 (也就是所有的 public 和 protected)。因此基类成员如果不想被派生类的成员函数访问，则应在基类中声明为 private：

访问 public protected private
同一个类 yes yes yes
派生类 yes yes no
外部的类 yes no no

同理，一个派生类会继承基类的所有方法，

  • 如果派生类没有显式定义，则会直接继承基类的构造函数，析构函数
  • 基类的友元函数不会被继承 ，友元函数本质不是成员函数；
